# v2 Go Project Layout

Standard Go convention. Module path: `github.com/gao-aipros/agentmemory`.

```
cmd/agentmemory/main.go      # entry point — wires dependencies, starts HTTP server
internal/
  handler/                    # HTTP handlers — thin layer, parse request, call service, write response
                              #   - REST handlers for hooks (/v1/api/*)
                              #   - Auth handlers (/v1/auth/*)
                              #   - MCP Streamable HTTP handler (/v1/mcp)
  service/                    # business logic — pipeline stages, search orchestration, consolidation
                              #   - observe, compress, summarize, consolidate, reflect
                              #   - smart-search orchestration
                              #   - context injection assembly
  store/                      # sqlc-generated DB access — type-safe Go functions from .sql queries
                              #   - generated by sqlc from internal/db/queries/
                              #   - never edit manually
  db/queries/                 # .sql query files — the source of truth for all database access
                              #   - organized by domain: observations.sql, memories.sql, sessions.sql, etc.
  mcp/                        # MCP tool registration — maps tool names to handler functions
                              #   - 51 v1 tools migrated
                              #   - new team management tools
                              #   - new auth key management tools
  hooks/                      # plugin hook scripts — shell scripts that call REST API
                              #   - pre-tool-use, post-tool-use, session-start, session-end, etc.
                              #   - delegates all logic to REST endpoints
  auth/                       # JWT + API key authentication
                              #   - token generation, validation, middleware
                              #   - TOTP setup and verification
  team/                       # team management logic
                              #   - CRUD, membership, role management
                              #   - visibility enforcement
  config/                     # env-var parsing — single source of truth for all configuration
migrations/                   # golang-migrate DDL files — numbered SQL migration files
                              #   - 001_initial_schema.up.sql, 001_initial_schema.down.sql
```

## Design Rationale

- `internal/` prevents external packages from importing internal implementation details
- `cmd/agentmemory/main.go` is the ONLY entry point — keeps binary build simple
- `store/` is generated code, never hand-edited — clear separation from hand-written `service/`
- `db/queries/` is the SQL source of truth — DBAs can review SQL without reading Go
- `mcp/` is separate from `handler/` because tool registration is a distinct concern from HTTP transport
- `hooks/` contains shell scripts, not Go code — they're the plugin surface that external agents invoke
